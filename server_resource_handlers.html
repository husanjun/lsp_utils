Server Resource Handlers¶

ServerStatus¶

class lsp_utils.ServerStatus¶

A ServerStatus enum for use as a return value from ServerResourceInterface.get_status().

UNINITIALIZED = 1¶

Initial status of the server.

ERROR = 2¶

Initiallation or update has failed.

READY = 3¶

Server is ready to provide resources.

ServerResourceInterface¶

class lsp_utils.ServerResourceInterface¶

An interface for implementating server resource handlers. Use this interface in plugins that manage their own server binary (GenericClientHandler.manages_server() returns True).

After implementing this interface, return an instance of implemented class from GenericClientHandler.get_server().

abstract needs_installation()¶

This is the place to check whether the binary needs an update, or whether it needs to be installed before starting the language server.

Return type

bool

Returns

True if the server needs to be installed or updated. This will result in calling install_or_update().

abstract install_or_update()¶

Do the actual update/installation of the server binary. Don’t start extra threads to do the work as everything is handled automatically.

Return type

None

abstract get_status()¶

Determines the current status of the server. The state changes as the server is being installed, updated or runs into an error doing those. Initialize with ServerStatus.UNINITIALIZED and change to either Set to ServerStatus.ERROR or ServerStatus.READY depending on if the server was installed correctly or is already installed.

Return type

int

Returns

A number corresponding to the ServerStatus class members.

abstract property binary_path: str¶

Returns a filesystem path to the server binary.

Return type

str

ServerPipResource¶

class lsp_utils.ServerPipResource(storage_path, package_name, requirements_path, server_binary_filename)¶

An implementation of lsp_utils.ServerResourceInterface implementing server management for pip-based servers. Handles installation and updates of the server in the package storage.

Parameters

  • storage_path (str) – The path to the package storage (pass lsp_utils.GenericClientHandler.storage_path())

  • package_name (str) – The package name (used as a directory name for storage)

  • requirements_path (str) – The path to the requirements.txt file, relative to the package directory. If the package LSP-foo has a requirements.txt file at the root then the path will be requirements.txt.

  • server_binary_filename (str) – The name of the file used to start the server.
